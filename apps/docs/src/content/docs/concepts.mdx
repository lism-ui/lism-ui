# コンセプト

Lism UI は以下のコンセプトを掲げています。

- ゼロビルドでどんな環境にも導入できる。
- 軽量なのに汎用性が高い。
- 車輪の再発明からの解放。
{/* - 方法論の提供である。 */}


## Concept 1. ゼロビルドでどんな環境でも使えるフレームワーク

Lismは、あくまでも**HTMLとCSSの設計が主体**になっており、特殊なビルド処理や複雑な設定ファイルは必要ありません。

{/* 最近流行りのゼロランタイムと逆行する、ゼロビルドです。 */}

<Alert preset="check">
CSSを読み込むだけで**素のHTMLサイト**や**WordPressサイトでも利用できる**ものになっています。
</Alert>

<Text c="content-2" mbs="10" fz="s">（一部、アコーディオンやタブの動作に関するJavaScriptが付属しています。）</Text>

ただし、**Lism Components**を使った方が圧倒的に開発体験は向上しますので、基本的には Vite や Next.js、Astro でのコンポーネントの活用を推奨しています。



## Concept 2. 軽量なのに、汎用性が高い。

ビルド不要のCSSフレームワークは他にもありますが、手軽に使用するために用意されている`~.all.css`みたいな全部入りCSSファイルを読み込むと容量がかなり大きくなってきます。


{/* ビルド環境を工夫できる場合は無駄なものを省くことができるので良いですが、そうでない場合は用意されているユーティリティクラスをすべて読み込む必要があります。 */}

{/* <Alert preset="check">
そこで、**用意している全CSS**を読み込んでも**あまり気にならない容量**に抑えることを意識しつつ、Lism設計だけでかなり柔軟なレイアウトを実現できるように意識しました。
</Alert> */}
{/* <Text mbs="20" fz="s">（現状、用途が限定的な拡張用コンポーネントのCSSを含めても、50kb程度）</Text> */}
{/* また、完全なユーティリティファースト設計ではないため、レイアウトモジュールに関するCSSについては分離して必要なページでのみ読み込むこともできます。 */}

{/* これにより、WordPressテーマ等の汎用性が必要な製品に取り込みやすくなっています。 */}

{/* ## Concept 3. なのに、柔軟である。 */}

特に、容量の面で一番の課題となるのが**レスポンシブ対応**です。

<Alert boxcolor="blue" icon={SmileyXEyes}>
汎用性を高めるために複数のブレイクポイントに対応したクラスを用意してしまうと、CSSの量が一気に肥大化してしまいます。
</Alert>

とはいえ、実際の制作現場ではブレイクポイントでのデザイン切り替えが必要となることが多いです。

<Alert boxcolor="green" icon='good'>
そこで、Lismでは**CSS変数とユーティリティクラスの組み合わせ方式**を考えました。  
CSSの容量を抑えつつ、**簡単にブレイクポイントごとの切り替えができる**ようになっています。
</Alert>

（デフォルトでブレイクポイントの切り替えに対応しているCSSプロパティの数は一部ですが、同じ形式に則ってCSSを追加することで、その他のプロパティについてもブレイクポイント切り替えが可能になります。）

また、軽量にすることを意識しつつも、可読性や利便性を犠牲にしすぎないことや、負荷が高い子孫セレクタの使用をできるだけ避ける、なども同時に意識しています。


## Concept 3. 車輪の再発明からの解放

Lism UI は CSSレベルのトークンやモジュール設計を提供するだけではなく、その設計に基づいた**ReactやAstroのコンポーネント**も提供しています。

さらに、アコーディオンやタブなど、どう実装するのが正解か非常に悩ましいコンテンツについても、個人的なベストプラクティスを詰め込んだ上で実装し、提供しています。(新たなベストプラクティスが出てきた場合は、随時更新していきます。)


<Alert boxcolor="green" icon='good'>
似たようなCSS、似たようなHTML、似たようなコンポーネントを何度も書くのはもうおしまいにしましょう。
</Alert>

柔軟性があり、軽量で、どんなプロジェクトにも導入できるフレームワークがここにあります。


## その他の特徴


### レイヤー階層・セレクタ優先度まで設計

デザイン的なモジュールクラスやトークン設計を提供するだけでなく、
CSSの階層構造をあらかじめ定義しています。

これに則ることにより、CSSで問題になりやすい詳細度の複雑化を軽減します。

### コンテナクエリの採用

Lismでは、メディアクエリではなくコンテナクエリをデフォルトで採用しています。  
そのため、慣れていない方は少し注意が必要です。

コンテナ要素内では position: fixed の挙動が変わったり、サイズ関係の挙動が少し変わったりすることがあります。


### Purge(Tree shaking)しやすいCSSの設計

ユーティリティクラスの活用に少し癖がありますが、PurgeCSS などを使って超軽量にできるようになっています。
`.-prop@bp`の形式で個別のプロパティごとに各ブレイクポイントに対応したクラスが1つずつあるのも、パージしやすくするためでもあります。


### ダークモードの実装も簡単

ダークモードの実装など、カラーテーマの切り替えも、ルートのトークンを切り替える(CSS変数の上書きする)だけで実装できます。
