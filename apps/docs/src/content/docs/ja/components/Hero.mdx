広いコンテンツエリアを作成するためのモジュールで、デフォルトでの単体使用では、ただの`isFullwide`な`Stack`です。

`<Hero>`内に`<HeroContent>`を使うことで、上・中央・下寄せでコンテンツを配置できます。をコントロールできるようになります。

{/* <p>親ブロックからbodyブロックを上・中央・下寄せをコントロールできるように。(marginを変える)</p>
<p>初期展開ブロックを選べるようにする。内部gridパターンも。</p> */}

## Props

- `<Hero>`
- `<HeroContent>`
	- `type`

## Import

```js
import { Hero, HeroContent } from '@loos/lism-core';
```

## Usage

`HeroContent` には`type`を指定でき、`top`、`center`、`bottom`のいずれかを指定することができます。  
それぞれのキーワードに合わせて出力されるpropsが変わります。

たとえば、`type='center'`の場合は、`my='auto'`, `hasGutter`, `isContainer='m'`が呼び出されます。

### HeroContent の子要素の高さに%を使う方法

{/* - l-stack(flex)の場合
	- Heroの高さを h でセット→ HeroContent は h も minH,maxH も fxb も % が使用できる.
	- Heroの高さを minH でセット → HeroContent は h も minH,maxH もfxb も % は無効になる。
	- Heroのアスペクト比をセット→ HeroContent は h も minH,maxH も fxb も % が使用できる.
		- minH, fxb を使うと、小さいウインドウではコンテンツに合わせてHeroの高さが可変する。
		- h,maxH を使うと、小さいウインドウではHeroContentの高さが外に溢れていく（ovy='auto'で内部スクロールできる）
- l-gridの場合
	- 何で高さを確保しても、HeroContent の高さは%が使える。
	- が、そのさらに子要素のhに%を使うには、結局 HeroContent をGrid にする必要がある。Layerを使ってもこれは一緒。
	（ HeroContent 自体を色々装飾するとpaddingの位置が問題になってくるので、HeroContentの子要素の高さに%をつかえるのが大事。）

→ 子要素に%を使うための複雑性はあまりかわらないので、その他の運用がシンプルになる stack の構成で決定。 */}

{/* -  */}

Hero 自体の高さを固定していい場合は、普通に Hero の高さを指定すれば子要素であるHeroContentや、そのさらに子要素のheightにも`%`を使うことができます。

<Preview>
	<PreviewTitle>普通に`h`を指定するケース</PreviewTitle>
	<PreviewArea resize>
		<Hero h='50vh' bgc='base-3'>
			<HeroContent type='center' minH='50%'>
				<Center minH='100%' bgc='base' bxsh='1'>
					<Text fw='bold' fz='l'>Height 50%</Text>
				</Center>
			</HeroContent>
		</Hero>
	</PreviewArea>
	<PreviewCode>
		```jsx "h='50vh'" "minH='50%'" "minH='100%'"
		<Hero h='50vh' bgc='base-3'>
			<HeroContent type='center' minH='50%'>
				<Center minH='100%' bgc='base' bxsh='1'>
					<Text fw='bold' fz='l'>Height 50%</Text>
				</Center>
			</HeroContent>
		</Hero>
		```
	</PreviewCode>
</Preview>


問題は、`<Hero>` の高さを minH やアスペクト比で指定したいケースです。  
(ウインドウ幅が広い時は特定の高さに制限したいが、ウインドウ幅が狭くなって内部のコンテンツ量がはみ出す時、自然と`Hero`の高さが伸びてくれるようなレイアウトを作りたいケース。)


基本的に、min-height を指定した要素の子要素では、heightに`%`を使っても無効となるため、一工夫必要です。

- `Hero` は `Stack` で構成されているので、`HeroContent`の`flex-grow`を`1`にして高さを伸ばします。
- 高さがある`grid`要素で、かつ`align-items`が`center`などの要素の直下では `height` の%指定が有効となるため、`HeroContent`を`<Grid>`で展開します。（`align-items`を指定せず、子要素を`my='auto'`や`aslf='center'`にしてもOKです。）


`type="stretch"`を指定すると上記の工程を手軽に呼び出せます。（`as={Grid} fxg='1' ai='center'` をセットする）


<Preview>

					<Text fw='bold' fz='xl'>min-height 50%</Text>

				</Center>
			</HeroContent>
			{/* <ShapeDivider shape='line2' c='#fff'/> */}
