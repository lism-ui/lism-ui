ナビメニューリストを作成できるコンポーネントです。

## Props

| プロパティ | 説明 |
|---|---|
| <PropBadge>`<NavMenu>`</PropBadge><br/>`layout` | 展開したい Lismコンポーネント を指定できます。 |
| <PropBadge>`<NavMenu>`</PropBadge><br/>`hovBgc` | `--hov-bgc`を出力します。（`.c--navMenu__link`に`.-hov:bgc:`を付けた時用）|
| <PropBadge>`<NavMenu>`</PropBadge><br/>`hovC` | `--hov-c`を出力します。 （`.c--navMenu__link`に`.-hov:c:`を付けた時用）|
| <PropBadge>`<NavMenuLink>`</PropBadge><br/>`hov` | hover時のスタイルを指定できます。デフォルトは`fade`が指定されています。 |
{/* | <PropBadge>`<NavMenu>`</PropBadge><br/>`nestLevel` | ネストの深さを指定します。`[data-nest-level]`に値を出力します。<br />ただし、トップレベルでは指定しないでください。（`:not([data-nest-level])`で変数の初期値をセットするため）| */}




## Import

```js
import { NavMenu } from '@lism-ui/core';
```

### ホバースタイルの調整

`<NavMenuLink>`からデフォルトで出力されるのは`-hov:fade`になっています。  
カラーを変化させたい場合、`<NavMenu>`（`.c--navMenu`）に`hovBgc`や`hovC`(`--hov-c`や`--hov-bgc`)を指定し、`<NavMenuLink>`（`.c--navMenu__link`）側で`-hov:c`や`-hov:bgc`を指定して受け取ります。

<Reference>
[hover（`hov`）の指定について](/ja/css/hover)
</Reference>

<Preview>
	<PreviewTitle>hover時のカラー指定</PreviewTitle>
	<PreviewArea p='30'>
		<NavMenu bd hasDivider hovBgc='text' hovC='base'>
			<NavMenuItem>
				<NavMenuLink href='#menu1' hov='c:,bgc:'>Menu item</NavMenuLink>
			</NavMenuItem>
			<NavMenuItem>
				<NavMenuLink href='#menu2' hov='c:,bgc:'>Menu item</NavMenuLink>
			</NavMenuItem>
			<NavMenuItem>
				<NavMenuLink href='#menu4' hov='c:,bgc:'>Menu item</NavMenuLink>
			</NavMenuItem>
		</NavMenu>
	</PreviewArea>
	<PreviewCode>
		```jsx "hovBgc" "hovC" "hov='c:,bgc:'"
		<NavMenu bd hasDivider hovBgc='text' hovC='base'>
			<NavMenuItem>
				<NavMenuLink href='#menu1' hov='c:,bgc:'>Menu item</NavMenuLink>
			</NavMenuItem>
			<NavMenuItem>
				<NavMenuLink href='#menu2' hov='c:,bgc:'>Menu item</NavMenuLink>
			</NavMenuItem>
			<NavMenuItem>
				<NavMenuLink href='#menu4' hov='c:,bgc:'>Menu item</NavMenuLink>
			</NavMenuItem>
		</NavMenu>
		```
	</PreviewCode>
</Preview>

変数をセットしなければ親から受け取れることを利用して、上の例では親の`<NavMenu>`にだけ変数をセットしています。  

各メニューリンク部分を以下のようにしても同じことができます。

```jsx
<NavMenuLink href='#menu1' hov={{c: 'base', bgc: 'text'}}>Menu item</NavMenuLink>
```



### 各リストのpaddingを一括で調整する方法

`<NavMenuLink>`(`.c--navMenu__link`)の`padding`は、`--pass-p`変数を受けとるようになっています。

これにより、`<NavMenu>`(`.c--navMenu`)側から`passVars.p`(`--pass-p`)をセットして各メニューリンク部分のpaddingを一括調整できます。

<Reference>
[pass-getについて](/ja/css/pass-get)
</Reference>

<Preview>
	<PreviewTitle>`--pass-p` でpaddingを一括指定</PreviewTitle>
	<PreviewArea p='30'>
		<NavMenu bd hasDivider passVars={{p:'30'}}>
			<NavMenuItem>
				<NavMenuLink href='#menu1'>Menu item</NavMenuLink>
			</NavMenuItem>
			<NavMenuItem>
				<NavMenuLink href='#menu2'>Menu item</NavMenuLink>
			</NavMenuItem>
			<NavMenuItem>
				<NavMenuLink href='#menu3'>Menu item</NavMenuLink>
			</NavMenuItem>
		</NavMenu>
	</PreviewArea>
	<PreviewCode>
		```jsx "passVars"
		<NavMenu bd hasDivider passVars={{p:'30'}}>
			<NavMenuItem>
				<NavMenuLink href='#menu1'>Menu item</NavMenuLink>
			</NavMenuItem>
			<NavMenuItem>
				<NavMenuLink href='#menu2'>Menu item</NavMenuLink>
		```

</Preview>

各`<NavMenuLink>`にpaddingを指定してももちろん大丈夫ですが、ネストしたメニューをアコーディオン化する場合などを想定し、`--pass-p`での管理が可能になっています。

### ネストしたメニューをアコーディオンで展開する

リンクのpaddingは`--pass-p`で管理されているため、アコーディオン化した時に `-get:p`クラスを使えば同じ量のpaddingを受け取れます。  


<Preview>
	<PreviewTitle>例</PreviewTitle>
	<PreviewArea p='30'>

			<NavMenuItem>
				<NavMenuLink hov='bgc:' href='#menu1'>Menu item</NavMenuLink>
			</NavMenuItem>
			<NavMenuItem>
				<NavMenuLink hov='bgc:' href='#menu2'>Menu item</NavMenuLink>
			</NavMenuItem>
			<NavMenuItem>
				<Accordion>
					<AccordionHeader hov='bgc:' get='p'>Menu item</AccordionHeader>
					<AccordionBody>
						<NavMenu pis='30'>
							<NavMenuItem>
								<NavMenuLink hov='bgc:' href='#child-menu1'>Nested item</NavMenuLink>
							</NavMenuItem>
							<NavMenuItem>
								<NavMenuLink hov='bgc:' href='#child-menu2'>Nested item</NavMenuLink>
							</NavMenuItem>
						</NavMenu>
					</AccordionBody>
				</Accordion>
			</NavMenuItem>
		</NavMenu>
	</PreviewArea>
