	"**Lism UI**" は、WEBサイトの骨組みをサクっと作れる軽量なフレームワークプロジェクトです。

Lism UI は以下のコンセプトを掲げています。

- ゼロビルドで素のHTMLサイトにも導入できる。
- 軽量で、かつ、汎用性も高い。

独自の**CSS設計理論**（**Lism CSS**）と、それを元にした**コンポーネント集**（**Lism Components**）を提供します。

入念に設計された Lism UI を学習することで、それ以降の車輪の再発明にかかる時間を削減できます。


<Callout boxcolor="green" icon='good'>

</Callout>

柔軟性で、軽量で、どんなプロジェクトにも導入できるフレームワークがここにあります。

## Lism CSS

Lism CSS では、WEBサイトデザインを以下の概念に分類します。

<Box p="box" bgc='base-2'>
	- Design Tokens
	- Layout State
	- Layouts
	- Objects
	- Prop Utilities
</Box>


※ [Every Layout](https://every-layout.dev/) + [Bootstrap](https://getbootstrap.com/) を 混ぜ合わせたようなイメージです。

WEBサイトで一般的によく使われるレイアウト・デザイン要素を細分化し、汎用的に使い回せるようになっています。

## Lism Components

Lism CSSを簡単に扱えるようにした専用のコンポーネントをReact,Astro形式で配布しています。

[Chakra UI](https://chakra-ui.com/) や [MUI](https://mui.com/) のような使い心地で、Lism CSSの設計思想に則ったモジュールを手軽に扱えるようになります。

## 特徴


### ゼロビルドでどんな環境でも使えるフレームワーク

Lismは、あくまでも**HTMLとCSSの設計が主体**になっており、特殊なビルド処理は必須ではありません。

{/* 最近流行りのゼロランタイムと逆行する、ゼロビルドです。 */}

<Callout type="check">
CSSを読み込むだけで**素のHTMLサイト**や**WordPressサイトでも利用できる**ものになっています。
</Callout>

<Text c="content-2" mbs="10" fz="s">（一部、アコーディオンやタブの動作に関するJavaScriptが付属しています。）</Text>

ただし、**Lism Components**を使った方が圧倒的に開発体験は向上しますので、基本的には Vite や Next.js、Astro 等の開発環境上でのコンポーネントの活用を推奨しています。


### CSS変数とユーティリティクラスの組み合わせ

Lism には、汎用性を確保しつつ、読み込むCSSを軽量に抑えるための工夫が詰まっています。
もちろん、軽量にすることを意識しつつも、可読性や利便性を犠牲にしすぎないことや、負荷が高い子孫セレクタの使用をできるだけ避ける、なども同時に意識しています。

CSSを読み込むだけで使えるフレームワークはいくつかありますが、手軽に使用するために用意されている`~.all.css`みたいな全部入りCSSファイルを読み込むと容量がかなり大きくなってきます。もしくは、CSSファイルが軽量でもHTMLのコード量が増えているだけだったり...。

{/* また、ビルド処理でCSSの読み込み量を最小限にしているものであっても、結局はHTML側の記述が膨大になったり、インラインで`<style>`タグが大量に挿入されていたりします。 */}

Lismでは、読み込むCSSの量とHTML側のコード量のどちらにも考慮し、バランスを考えつつ設計しています。  
また、軽量にすることを意識しつつも、可読性や利便性を犠牲にしすぎないことや、負荷が高い子孫セレクタの使用をできるだけ避ける、なども同時に意識しています。


特に、容量の面で一番の課題となるのは**レスポンシブ対応**です。


{/* ビルド環境を工夫できる場合は無駄なものを省くことができるので良いですが、そうでない場合は用意されているユーティリティクラスをすべて読み込む必要があります。 */}

{/* <Callout type="check">
そこで、**用意している全CSS**を読み込んでも**あまり気にならない容量**に抑えることを意識しつつ、Lism設計だけでかなり柔軟なレイアウトを実現できるように意識しました。
</Callout> */}
{/* <Text mbs="20" fz="s">（現状、用途が限定的な拡張用コンポーネントのCSSを含めても、50kb程度）</Text> */}
{/* また、完全なユーティリティファースト設計ではないため、レイアウトモジュールに関するCSSについては分離して必要なページでのみ読み込むこともできます。 */}

{/* これにより、WordPressテーマ等の汎用性が必要な製品に取り込みやすくなっています。 */}

{/* ### Concept 3. なのに、柔軟である。 */}



<Callout boxcolor="blue" icon={SmileyXEyes}>
汎用性・利便性を高めるために、すべてのユーティリティクラスをブレイクポイントに対応してしまうと、CSSの量が一気に肥大化してしまいます。
</Callout>

[Every Layout](https://every-layout.dev/)のような、ブレイクポイントに依存しない優秀なデザインシステムも世の中にはありますが、実際の制作現場ではまだまだブレイクポイントでのデザイン切り替えが必要となることが多いです。

そこで、Lismでは**CSS変数とクラスの組み合わせ方式**を考えました。  

<Callout boxcolor="green" icon='good'>
CSSの容量を抑えつつ、HTML側のソースコード量の肥大化も少なく、主要なプロパティに関しては**簡単にブレイクポイントごとの切り替えができる**ようになっています。
</Callout>

これまで見たことのない面白いアプローチを思いつけたと思っています。

（すべてのCSSプロパティがブレイクポイントの切り替えに対応しているわけではありませんが、sassファイルから設定を上書きするか、同じ形式に則ってCSSを追加することで、任意のプロパティについてもブレイクポイント対応が可能になります。）

{/* 
### 車輪の再発明からの解放

Lism UI は CSSレベルのトークンやモジュール設計を提供するだけではなく、その設計に基づいた**ReactやAstroのコンポーネント**も提供しています。

さらに、アコーディオンやタブなど、どう実装するのが正解か非常に悩ましいコンテンツについても、個人的なベストプラクティスを詰め込んだ上で実装し、提供しています。(新たなベストプラクティスが出てきた場合は、随時更新していきます。)
 */}


### その他の特徴


#### コンテナクエリの採用

Lismでは、メディアクエリではなくコンテナクエリをデフォルトで採用しています。  
そのため、慣れていない方は少し注意が必要です。

コンテナ要素内では position: fixed の挙動が変わったり、サイズ関係の挙動が少し変わったりすることがあります。

#### レイヤー階層・セレクタ優先度まで設計

デザイン的なモジュールクラスやトークン設計を提供するだけでなく、
CSSの階層構造をあらかじめ定義しています。

これに則ることにより、CSSで問題になりやすい詳細度の複雑化を軽減します。



{/* #### Purge(Tree shaking)しやすいCSSの設計

ユーティリティクラスの活用に少し癖がありますが、PurgeCSS などを使って超軽量にできるようにもなっています。
`.-prop@bp`の形式で個別のプロパティごとに各ブレイクポイントに対応したクラスが1つずつあるのも、パージしやすくするためでもあります。 */}


{/* #### ダークモードの実装も簡単

ダークモードの実装など、カラーテーマの切り替えも、ルートのトークンを切り替える(CSS変数の上書きする)だけで実装できます。 */}

{/* ### WordPressフレンドリー

Lism は、WordPressにも導入しやすいように設計されており、Lism設計に基づいたブロックテーマも開発予定です。

- 軽量
- WordPress側のクラス名と衝突しない命名規則
- Tokenを活用する点が、`theme.json` 関連機能との親和性が高い。
- コンテンツ幅の制御手法などもWordPressに合わせた設計が可能。
- Reactベースのコンポーネントがあるため、ブロック化しやすい。
- Lismベースのブロックでコンテンツを管理すれば、WordPressをヘッドレスCMSとして使いやすくなる。(サイト全体のCSSとコンテンツに必要なCSSがLismに集約されるため) */}
